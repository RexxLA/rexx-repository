The UTF-8 encoding class

License: Apache License 2.0 [ https://www.apache.org/licenses/LICENSE-2.0 ]

Description

The UTF8 class implements encoding/decoding from/to the UTF8 format. This encoder/decoder is, in a sense, somewhat special: since UTF8 is the designated interchange format for Unicode Rexx strings (i.e., how the string presents itself, although not necessarily how it is internally represented), the encode method has little to do, apart from ensuring that the result is of the desired class.

This is a subclass of the Encoding class. Please refer to the documentation for that class for additional details.

  • decode CLASS

    The .UTF8~decode method

    Decode takes a UTF8 string as an argument, and returns a choice of Unicode conversions of the string (UTF8, UTF32, or both), according to the third, optional format parameter (by default, it returns the UTF8 version only).

    The argument string is checked for UTF8 well-formedness. The behaviour of the decode method when string is not well-formed depends on the value of its second, optional, option argument.

    When option is "Syntax"", if string is ill-formed, a Syntax condition is raised.

    When option is the null string, if string is ill-formed, the null string is returned.

    Both option and format are case-insensitive.

    param: string The string to decode.
    param: option = "" What to do if we find an encoding error. "Syntax" produces a syntax error, and "" (the default) returns the null string.
    param: format = "UTF8" Format may be the null string, "UTF8" or "UTF-8", in which case a UTF-8 version of the argument string is returned; it can be "UTF-32" or "UTF-32", in which case a UTF-32 version of the string is returned; or it can be any combination of blank-separated values (repetitions are allowed). If both UTF-8 and UTF-32 versions are requested, the returned value is an array containing the UTF-8 and the UTF-32 versions of the argument string (in this order).
    condition: Syntax 93.900 Invalid option 'option'.
    condition: Syntax 93.900 Invalid format 'format'.
    condition: Syntax 23.900 Invalid UTF-8 sequence in position n of string: 'hex-value'X.
    see: encode


    USE ARG string, format = "", errorHandling = ""
  • encode CLASS

    The .UTF8~encode method

    The encode method of the UTF8 class is special, in the sense that UTF8 is, precisely, the encoding we are using as an interchange format. Because of that reason, if an encoding (to UTF-8) of a Text or Codepoints string string is requested, the result is automatically the result of calling the makeString method (Text and Codepoints are sanitized on construction, and therefore the string is well-formed by definition).

    If the string argument is a simpler string (e.g., a Bytes string), then it will have to be decoded first (i.e., checked for UTF-8 well-formedness), and then we can return this very same string.

    Please note that the present implementation supports only UTF-8 as the encoding of source programs. If other encodings were to be supported, the preliminary decoding and sanitizing should, of course, use the corresponding decoder.

    param: string The string we want to encode
    param: option What to do if we find an encoding error. "Syntax" raises a syntax condition, and "" (the default) indicates the error by returning the null string.
    condition: Syntax 93.900 Invalid option 'option'.
    condition: Syntax 23.900 Invalid UTF-8 sequence in position n of string: 'hex-value'X.
    see: decode
